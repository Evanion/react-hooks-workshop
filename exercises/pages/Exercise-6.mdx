# Ex. 6 - Render props

## Instructions

L'application gère les erreurs, mais pas de manière fine. Le comportement souhaité serait que l'erreur soit affichée à la place de la liste de films et non sur toute l'application.

La stratégie adoptée est donc d'isoler la liste de films et le fetching des données dans un composant afin de pouvoir l'englober dans un `Catch`.

- Créer un composant "render props" dans `MovieSearch` dans `src/containers/MovieDb.js`
- Intégrer ce composant dans `App` et ajouter un `Catch` autour

### Résultat attendu

Lorsque le champs de recherche est vide, j'obtiens une erreur à la place de la liste de films.

## Aide

### Render props : implémentation

```js
export function Counter({ initialValue, children }) {
  const count = useCounter(initialValue)
  return children(count)
}
```

### Render props : utilisation

```js
function Example() {
  return <Counter>{count => <div>{count}</div>}</Counter>
}
```
